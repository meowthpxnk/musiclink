# Структура папки tracks

## Как добавить новый трек:

1. Добавьте обложку в папку `covers/` (например, `my_track.jpg`)

2. Добавьте запись в файл `data.json`:
```json
{
  "id": "my_track",
  "title": "Название трека",
  "artist": "Имя исполнителя",
  "cover_file": "my_track.jpg",
  "track_url": "https://ссылка-на-трек.mp3",
  "description": "Описание трека (необязательно)",
  "platforms": {
    "vk": "",
    "yandex_music": "",
    "spotify": "",
    "apple_music": "",
    "youtube_music": ""
  }
}
```

3. Перезапустите приложение или оно автоматически подхватит изменения

## Формат данных в data.json:

### Глобальные платформы (`global_platforms`):
Ссылки на платформы, которые будут использоваться для всех треков, если у конкретного трека не указана своя ссылка:
- `vk` - ссылка на VK
- `yandex_music` - ссылка на Яндекс Музыку
- `spotify` - ссылка на Spotify
- `apple_music` - ссылка на Apple Music
- `youtube_music` - ссылка на YouTube Music

### Поля трека:
- `id` - уникальный идентификатор трека (используется в URL, например `/my_track`)
- `title` - название трека
- `artist` - исполнитель
- `cover_file` - имя файла обложки из папки `covers/` (если пусто, будет показано "TBD")
- `track_url` - ссылка на аудиофайл или страницу с треком
- `description` - описание трека (необязательное поле)
- `platforms` - объект со ссылками на платформы для конкретного трека:
  - Если ссылка указана - используется она
  - Если пустая строка - используется глобальная ссылка из `global_platforms`
  - Если и глобальная пустая - кнопка не отображается

**Приоритет:** ссылка трека > глобальная ссылка > кнопка не показывается
